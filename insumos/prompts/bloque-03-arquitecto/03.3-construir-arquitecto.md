# Prompt 3.3 — Construir el archivo del agente

**Bloque 3 · Agente arquitecto**

```
CONTEXTO:
Tenemos la spec final del agente `arquitecto` después de iteración manual.
El formato de un agente de Claude Code es un archivo markdown con:
- Encabezado YAML con `name`, `description`, y opcionalmente `tools`.
- Cuerpo en markdown que funciona como system prompt del agente.

OBJETIVO:
Generar el archivo `.claude/agents/arquitecto.md` que cumple la spec
acordada.

RESTRICCIONES:
- El `description` del YAML debe ser claro respecto a cuándo Claude
  debe invocar este agente automáticamente.
- El system prompt fuerza al agente a:
  1. Leer spec.md y AGENTS.md antes de proponer nada.
  2. Proponer mínimo 2 alternativas por decisión arquitectónica.
  3. Para cada alternativa, dar trade-offs concretos (no "es más
     rápido", sino "esta opción requiere X y a cambio te da Y").
  4. Nunca decidir por el humano. Siempre cerrar con: "¿cuál eliges?".
  5. Si la spec tiene huecos bloqueadores, listarlos antes de proponer
     y detenerse.
- En español.

CRITERIO DE ÉXITO:
Archivo creado, válido como agente de Claude Code, y al invocarlo
sobre la spec del proyecto produce el output que la spec del agente
describe.
```

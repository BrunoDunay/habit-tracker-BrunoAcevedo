# Prompt 3.1 — Spec del agente arquitecto

**Bloque 3 · Agente arquitecto**

> Trabaja en la rama `feat/agente-arquitecto` (desde `develop`). El commit se hace al cerrar el Prompt 3.4.

```
CONTEXTO:
Voy a crear un agente custom de Claude Code llamado `arquitecto` que
vivirá en `.claude/agents/arquitecto.md`. Su trabajo: tomar la spec
del proyecto Habit Tracker y proponer Architecture Decision Records
(ADRs) con trade-offs reales, no decisiones de "porque sí".

El developer que va a usar este agente tiene experiencia básica en
desarrollo pero no en arquitectura. No domina trade-offs entre, por
ejemplo, Server Components vs Client Components en Next.js 15, ni
entre RLS de Supabase vs middleware en Next.js. El agente debe darle
opciones con trade-offs claros para que él decida.

Principio operativo del proyecto: los agentes proponen, el humano
decide.

OBJETIVO:
Escribir la spec del agente `arquitecto` aplicando los 4 elementos:
objetivo, scope, criterios de aceptación verificables, no-goals.

RESTRICCIONES:
- La spec es para el agente, no para la app. Describe qué hace el
  agente, no qué hace el producto.
- Criterios de aceptación verificables: cuando le pase una spec al
  agente, ¿cómo sé si su output cumple?
- No-goals explícitos. Mínimo 3. Ejemplos: "no decide por el humano",
  "no implementa código", "no escribe el ADR final, solo propone
  contenido".
- Máximo 50 líneas.

CRITERIO DE ÉXITO:
Otro developer podría tomar esta spec y construir el agente sin
contexto adicional. Y al recibir el output del agente, sabría si
cumple o no.
```

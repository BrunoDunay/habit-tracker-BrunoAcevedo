# Prompt 11.1 — Generar plan.md inicial

**Bloque 11 · Plan.md y readiness de entorno**

> Trabaja en la rama `docs/plan-inicial` (desde `develop`). El commit se hace al cerrar el Prompt 11.3.

```
CONTEXTO:
El repo tiene:
- spec.md (qué construir)
- docs/adr/0001, 0002, 0003 (decisiones arquitectónicas)
- docs/diseño.md (sistema visual y componentes)
- docs/pruebas-manuales.md (qué validar al final)
- AGENTS.md (contrato del proyecto)

OBJETIVO:
Generar `plan.md` con las primeras 10-15 tareas atómicas en orden
de implementación. Cada tarea es lo que el agente `implementer`
va a ejecutar.

RESTRICCIONES:
- Cada tarea ejecutable en menos de 1 hora.
- Cada tarea tiene criterio de hecho verificable: una línea que
  diga cómo sé que terminó.
- Cada tarea referencia, cuando aplica:
  - El ADR que la justifica.
  - La prueba manual que la valida.
- Ninguna tarea depende de más de 2 tareas previas.
- Las primeras 2 tareas son setup técnico (inicializar Next.js,
  instalar y configurar cliente de Supabase). Luego viene el modelo
  de datos. Luego auth. Luego el primer CRUD.
- No incluir tareas de la extensión todavía — solo núcleo.

CRITERIO DE ÉXITO:
Puedo ejecutar la tarea 1 ahora mismo sin más contexto. Y al terminar
la tarea 1, puedo verificar que terminó.
```

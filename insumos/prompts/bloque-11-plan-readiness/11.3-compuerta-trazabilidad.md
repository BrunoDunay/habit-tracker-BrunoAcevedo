# Prompt 11.3 — Compuerta de trazabilidad: plan vs ADRs y pruebas

**Bloque 11 · Plan.md y readiness de entorno**

```
CONTEXTO:
Tengo plan.md con tareas atómicas, los ADRs en docs/adr/ y las pruebas
en docs/pruebas-manuales.md.

OBJETIVO:
Verificar que el plan es coherente con los ADRs y las pruebas. Solo
reporte, no reescribas el plan.

RESTRICCIONES:
- Para cada tarea: di qué ADR la justifica y qué prueba la valida (si
  aplica).
- Marca cualquier tarea que CONTRADIGA un ADR (ejemplo típico: un ADR
  decidió cliente + SWR y una tarea asume Server Actions).
- Marca tareas sin criterio de hecho verificable.

CRITERIO DE ÉXITO:
Una tabla tarea → ADR → prueba. Cero contradicciones con los ADRs, o
sé exactamente qué tarea corregir antes de construir.
```

> Solo lectura. Es la compuerta más importante: una contradicción tarea-vs-ADR se corrige a mano ahora, no en el build.

**Rama y commit:** `docs/plan-inicial` desde `develop` → `docs: plan.md inicial con tareas atómicas` → merge a `develop`.

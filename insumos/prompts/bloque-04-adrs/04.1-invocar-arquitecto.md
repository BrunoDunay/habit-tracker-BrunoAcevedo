**Bloque 4 · Generar ADRs**

> Trabaja en la rama `docs/adrs-iniciales` (desde `develop`). El commit se hace al cerrar el Prompt 4.3.

```
CONTEXTO:
La spec actual del proyecto Habit Tracker está en `spec.md`. El contrato
del proyecto está en `AGENTS.md`. Quiero proceder a las decisiones
arquitectónicas iniciales.

OBJETIVO:
Recibir propuestas de ADR para: modelo de datos, autenticación,
frontera cliente/servidor. Y cualquier decisión adicional que el
arquitecto considere bloqueante para empezar el build.

RESTRICCIONES:
- El arquitecto no decide. Yo decido.
- Si mi spec tiene huecos bloqueantes, dímelos antes de proponer.

CRITERIO DE ÉXITO:
Salgo con propuestas suficientes para escribir 3 ADRs.
```

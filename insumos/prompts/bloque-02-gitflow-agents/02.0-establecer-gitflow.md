# Prompt 2.0 — Establecer el flujo de trabajo con gitflow

**Bloque 2 · Gitflow y AGENTS.md del proyecto**

```
CONTEXTO:
El repo ya está inicializado con los commits del Bloque 0 en la rama
main. A partir de ahora el proyecto sigue gitflow: main estable,
develop como rama de integración, y una rama tipada por cada unidad
de trabajo.

OBJETIVO:
Crear la rama develop desde main y dejar el repo listo para trabajar
con ramas. No crear código ni archivos nuevos.

RESTRICCIONES:
- Crea la rama develop a partir de main y cámbiate a ella.
- No borres ni reescribas los commits que ya existen en main.
- Reporta en qué rama quedaste parado.

CRITERIO DE ÉXITO:
`git branch` muestra main y develop. Estoy parado en develop.
```

> No crea commit: crea la rama `develop`. De aquí en adelante cada bloque se trabaja en su rama tipada (ver tabla en el [README](../README.md)) y se mergea a `develop` al cerrar.

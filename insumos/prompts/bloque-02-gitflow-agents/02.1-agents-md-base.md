# Prompt 2.1 — Generar AGENTS.md base

**Bloque 2 · Gitflow y AGENTS.md del proyecto**

> Trabaja en la rama `chore/agents-md` (desde `develop`). El commit se hace al cerrar el Prompt 2.2.

```
CONTEXTO:
Proyecto Habit Tracker. Stack: Next.js 15 con App Router, Supabase
(Postgres + Auth + Storage), Vercel para deploy, TypeScript en modo
estricto, Tailwind para estilos. El proyecto se construye dirigiendo
agentes custom: arquitecto, diseñador, qa, reviewer e implementer.
Todos los agentes deben respetar el mismo contrato.

Reglas operativas innegociables del proyecto:
1. No se escribe código sin plan aprobado.
2. Commits atómicos y verificables: uno por unidad funcional, no
   commits "implement everything".
3. Cualquier edición manual de código (no generada por un agente)
   se documenta en CONTEXT.md con justificación.
4. Todo el trabajo se versiona con gitflow: nada se queda sin
   commitear. main estable, develop de integración, y una rama
   tipada (feat/, docs/, chore/, fix/) por unidad de trabajo que se
   mergea a develop.

OBJETIVO:
Generar AGENTS.md que sirva como contrato del proyecto, leído por
todos los agentes y por cualquier developer humano que se incorpore.

RESTRICCIONES:
- Máximo 100 líneas.
- Secciones obligatorias: stack, convenciones de TypeScript, estructura
  de carpetas esperada, política de commits, flujo git (gitflow: ramas
  main/develop, ramas tipadas feat/ docs/ chore/ fix/, merge a develop),
  regla de CONTEXT.md, prohibiciones explícitas (uso de `any` sin
  justificación, librerías de componentes pesadas como Material UI o
  Chakra, tests automatizados para este alcance).
- En español.
- Sin emojis.

CRITERIO DE ÉXITO:
Otro agente que lo lea sabe qué decisiones puede tomar libremente y
cuáles están cerradas por contrato.
```

# Prompt 1.2 — Crítica desde 3 perspectivas

**Bloque 1 · Crítica de la spec inicial**

```
CONTEXTO:
Misma spec de `insumos/spec-borrador.md`. Ya hiciste inventario en el
prompt anterior.

OBJETIVO:
Criticar la spec desde 3 perspectivas distintas. No la reescribas. Solo
lista de problemas concretos.

RESTRICCIONES:
- Perspectiva 1: un arquitecto que tiene que diseñar el modelo de datos
  a partir de esta spec. ¿Qué información le falta para decidir tablas,
  columnas, relaciones?
- Perspectiva 2: QA que tiene que escribir pruebas manuales para validar
  la app cuando esté hecha. ¿Qué criterios de aceptación son inverificables
  y por qué?
- Perspectiva 3: un developer que se incorpora al proyecto sin contexto
  previo y tiene que implementar la app a partir de esta spec. ¿Qué
  decisiones tendría que improvisar porque no están escritas?

CRITERIO DE ÉXITO:
3 listas, una por perspectiva. Cada problema en una sola línea, sin
párrafos. Mínimo 3 problemas por perspectiva.
```

> Solo lectura: este paso no crea rama ni commit.

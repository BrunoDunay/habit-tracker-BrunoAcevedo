# Prompt 1.1 — Inventario de la spec

**Bloque 1 · Crítica de la spec inicial**

```
CONTEXTO:
El proyecto Habit Tracker tiene una spec inicial en `insumos/spec-borrador.md`.
El brief del producto exige: app web, autenticación email/password, CRUD de
hábitos (nombre, descripción, frecuencia diaria o semanal), check-in diario,
vista de progreso con racha actual, y exactamente una extensión funcional
del catálogo definido en el brief. Stack obligatorio: Next.js 15 con App
Router, Supabase para Postgres y Auth, Vercel para deploy.

OBJETIVO:
Hacer inventario de qué cubre la spec y qué no cubre. Sin emitir juicio.
Solo mapeo.

RESTRICCIONES:
- No reescribas nada.
- No propongas mejoras.
- No me digas si está buena o mala.
- Para cada uno de los 4 elementos de la spec (objetivo, scope, criterios
  de aceptación, no-goals): di si está presente, parcial o ausente, y en
  una línea qué dice.

CRITERIO DE ÉXITO:
Una tabla de 4 filas. Yo, leyéndola, puedo ubicarme en la spec sin haberla
leído entera.
```

> Solo lectura: este paso no crea rama ni commit.

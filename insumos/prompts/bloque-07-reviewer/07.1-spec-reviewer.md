# Prompt 7.1 — Spec del agente reviewer

**Bloque 7 · Agente reviewer**

> Trabaja en la rama `feat/agente-reviewer` (desde `develop`). El commit se hace al cerrar el Prompt 7.3.

```
CONTEXTO:
Voy a crear el agente `reviewer` en `.claude/agents/reviewer.md`. Su
trabajo: revisar un diff o commit reciente del proyecto y reportar
problemas contra AGENTS.md, contra la spec y los ADRs, y contra las
reglas operativas del proyecto.

OBJETIVO:
Escribir la spec del agente con los 4 elementos.

RESTRICCIONES:
- El reviewer reporta, no arregla. Lista problemas, no parchea código.
- Cubre 4 ejes:
  1. ¿Respeta AGENTS.md? (TypeScript estricto, convenciones, sin
     `any` injustificado, gitflow, etc.)
  2. ¿El commit es atómico? (¿hace una sola cosa o mezcla varias?)
  3. ¿Está alineado con la spec y los ADRs? (¿implementa lo prometido
     y solo lo prometido?)
  4. ¿Tiene rastros de "TODO", "fix later", console.log de debug?
- No-goals: "no aplica fixes", "no mergea PRs", "no evalúa estética
  del código si las convenciones de AGENTS.md no la cubren".
- Máximo 50 líneas.

CRITERIO DE ÉXITO:
Al usar el reviewer sobre un commit, se recibe una lista priorizada
de problemas accionables.
```

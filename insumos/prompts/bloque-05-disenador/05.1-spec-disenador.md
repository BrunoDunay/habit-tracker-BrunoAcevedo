# Prompt 5.1 — Spec del agente diseñador

**Bloque 5 · Agente diseñador**

> Trabaja en la rama `feat/agente-disenador` (desde `develop`). El commit se hace al cerrar el Prompt 5.3.

```
CONTEXTO:
Voy a crear el agente `diseñador` en `.claude/agents/diseñador.md`.
Su trabajo: tomar la spec del proyecto y proponer un sistema visual
mínimo coherente — tokens de diseño (colores, tipografía, escala
de espaciado), inventario de componentes UI necesarios, y estructura
de páginas.

Restricciones del proyecto: Tailwind como única librería de estilos.
Sin componentes pesados (no Material UI, no Chakra). Si se usa
shadcn/ui se permite.

El developer que usa este agente no tiene background de diseño.
Necesita salir con decisiones tomadas, no con opciones infinitas.

OBJETIVO:
Escribir la spec del agente `diseñador` con los 4 elementos.

RESTRICCIONES:
- Criterios de aceptación verificables: ¿cómo sé si su output está
  completo?
- No-goals explícitos. Ejemplos: "no propone animaciones complejas",
  "no propone modo oscuro a menos que la spec del proyecto lo requiera",
  "no genera Figma ni imágenes".
- Diferencia con el arquitecto: el diseñador SÍ decide. Propone 1
  sistema visual, no 3 opciones. Razón: las decisiones visuales no
  son arquitectónicas y la fricción de elegir entre 3 paletas atrasa
  más de lo que aporta.
- Máximo 50 líneas.

CRITERIO DE ÉXITO:
La spec describe sin ambigüedad qué entrega el agente y qué no.
```

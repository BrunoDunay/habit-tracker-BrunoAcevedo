# Prompt 13.3 — Generar CONTINUAR.md (el mapa del build)

**Bloque 13 · Cierre: Build-Ready y cómo continuar**

```
CONTEXTO:
El andamiaje está completo y BUILD-READY.md define cuándo arrancar. El
build se hará dirigiendo al agente implementer tarea por tarea desde
plan.md, bajo gitflow.

OBJETIVO:
Generar CONTINUAR.md: la guía paso a paso para empezar y sostener el
build a partir de aquí.

RESTRICCIONES:
- Describe el loop de build: crear rama feat/ desde develop por tarea
  de plan.md, invocar al implementer, revisar con el reviewer, commit
  atómico, merge a develop. Una tarea = una rama = un commit.
- Incluye qué hacer si el implementer se atora 2 veces (edición manual
  documentada en CONTEXT.md).
- Incluye cuándo se usan release/ (al desplegar a Vercel) y hotfix/.
- En español, máximo 40 líneas.

CRITERIO DE ÉXITO:
Otro developer abre CONTINUAR.md y sabe exactamente cuál es el primer
comando y el loop a repetir hasta terminar el build.
```

**Rama y commit:** `docs/build-ready` desde `develop` → commits `docs: BUILD-READY.md` y `docs: CONTINUAR.md` → merge a `develop`.

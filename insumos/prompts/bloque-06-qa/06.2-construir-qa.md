# Prompt 6.2 — Construir el agente qa

**Bloque 6 · Agente qa**

```
CONTEXTO:
Spec del agente `qa` finalizada.

OBJETIVO:
Materializar `.claude/agents/qa.md`.

RESTRICCIONES:
- YAML válido con name, description, tools.
- System prompt fuerza al agente a:
  1. Leer spec.md, AGENTS.md y los ADRs antes de generar pruebas.
  2. Para cada criterio de aceptación de la spec, generar 1 prueba
     manual con: precondición, pasos numerados, resultado esperado.
  3. Si un criterio es inverificable, devolver "criterio X no es
     verificable porque..." y detenerse en ese criterio.
  4. Output en formato markdown apto para guardar como
     `docs/pruebas-manuales.md`.
- En español.

CRITERIO DE ÉXITO:
Agente creado. Output ejecutable paso a paso por una persona sin
contexto previo del proyecto.
```

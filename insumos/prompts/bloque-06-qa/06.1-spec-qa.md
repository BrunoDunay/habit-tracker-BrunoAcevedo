# Prompt 6.1 — Spec del agente qa

**Bloque 6 · Agente qa**

> Trabaja en la rama `feat/agente-qa` (desde `develop`). El commit se hace al cerrar el Prompt 6.4.

```
CONTEXTO:
Voy a crear el agente `qa` en `.claude/agents/qa.md`. Su trabajo:
tomar la spec del proyecto y los ADRs, y generar un plan de pruebas
manuales — una prueba por criterio de aceptación de la spec.

En este proyecto no se contemplan tests automatizados. Solo pruebas
manuales descritas en markdown, ejecutables paso a paso por una
persona.

OBJETIVO:
Escribir la spec del agente `qa` con los 4 elementos.

RESTRICCIONES:
- Cada prueba manual debe ser ejecutable por una persona sin
  ambigüedad: pasos numerados, datos de entrada explícitos, resultado
  esperado observable.
- El agente debe rechazar criterios de aceptación inverificables y
  reportar qué les falta para hacerlos verificables.
- No-goals: "no genera tests automatizados", "no propone herramientas
  de testing", "no diseña casos exploratorios — solo cubre criterios
  de la spec".
- Máximo 50 líneas.

CRITERIO DE ÉXITO:
La spec deja claro qué entrega el agente y qué rechaza.
```

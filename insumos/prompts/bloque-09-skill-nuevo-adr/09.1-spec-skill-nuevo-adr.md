# Prompt 9.1 — Spec del skill nuevo-adr

**Bloque 9 · Skill nuevo-adr**

> Trabaja en la rama `feat/skill-nuevo-adr` (desde `develop`). El commit se hace al cerrar el Prompt 9.2.

```
CONTEXTO:
Voy a crear un skill custom en `.claude/skills/nuevo-adr/SKILL.md`.
Su trabajo: cuando alguien (un humano o el agente arquitecto) quiere
registrar una nueva decisión arquitectónica, este skill garantiza que
el ADR resultante tenga formato estandarizado y calidad mínima.

Formato canónico de ADR:
- Número correlativo
- Título corto y descriptivo
- Estado (propuesto, aceptado, deprecado)
- Contexto
- Decisión
- Alternativas consideradas (mínimo 1, con trade-offs reales)
- Consecuencias (mínimo 1 negativa o trade-off aceptado)

OBJETIVO:
Escribir la spec del skill con los 4 elementos.

RESTRICCIONES:
- El skill es un procedimiento, no un rol. No "piensa". Solo garantiza
  formato y completitud.
- Debe rechazar ADRs sin alternativa real ("alternativa: no hacer nada"
  no cuenta como alternativa).
- Debe rechazar ADRs sin consecuencia negativa explícita.
- Auto-incrementa el número del ADR mirando los archivos existentes
  en `docs/adr/`.
- No-goals: "no propone decisiones", "no juzga si la decisión es buena",
  "no edita ADRs existentes".

CRITERIO DE ÉXITO:
Spec lista para construir el skill.
```

# Prompt 0.2 — Borrador rápido de spec a partir del brief

**Bloque 0 · Génesis del proyecto**

```
CONTEXTO:
Tengo el brief del producto en `insumos/brief.md`. Quiero un primer
borrador de spec.md para empezar a iterar. No necesito que sea
perfecto — necesito que exista para tener algo que criticar y mejorar.

OBJETIVO:
Generar un primer borrador de spec.md con los 4 elementos canónicos:
Objetivo, Scope (sí entra / no entra), Criterios de aceptación,
No-goals.

RESTRICCIONES:
- Lee insumos/brief.md y genera la spec a partir de ahí.
- No me hagas preguntas. Si algo del brief es ambiguo, toma la
  interpretación más sencilla y sigue.
- Máximo 60 líneas.
- En español.

CRITERIO DE ÉXITO:
spec.md existe con los 4 elementos visibles. No necesariamente bien
hechos — solo presentes.
```

> Este borrador saldrá flojo a propósito (las restricciones "no me hagas preguntas" y "toma la interpretación más sencilla" lo fuerzan). Es material para la crítica del Bloque 1. El commit del borrador se hace en el Prompt 0.3.

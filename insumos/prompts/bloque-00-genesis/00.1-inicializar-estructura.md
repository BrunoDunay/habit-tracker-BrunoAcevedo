# Prompt 0.1 — Inicializar la estructura del proyecto

**Bloque 0 · Génesis del proyecto**

```
CONTEXTO:
Estoy arrancando un proyecto nuevo llamado Habit Tracker. Stack
definido: Next.js 15 con App Router, Supabase para Postgres y Auth,
Vercel para deploy, TypeScript estricto, Tailwind. Voy a construir
el proyecto dirigiendo agentes custom de Claude Code, así que la
estructura del repo necesita acomodar tanto el código como los agentes
y skills, y la documentación de proceso (spec, ADRs, diseño, pruebas,
plan, contexto).

El código de Next.js todavía NO se inicializa en este momento — eso
viene después, cuando exista plan.md. Por ahora solo la estructura
de documentación y configuración de Claude Code.

OBJETIVO:
Crear los archivos y carpetas vacíos (o con contenido mínimo) que
forman la estructura inicial del proyecto. Inicializar git con un
primer commit.

RESTRICCIONES:
- Archivos a crear vacíos: spec.md, AGENTS.md, CONTEXT.md.
- Carpetas a crear vacías: .claude/agents/, .claude/skills/,
  docs/adr/, insumos/.
- README.md con: nombre del proyecto, una línea de descripción,
  stack, estado actual ("documentación en construcción").
- .gitignore con: node_modules/, .env*, .DS_Store, .next/, dist/.
- Mover el archivo brief.md que ya existe en la raíz a insumos/brief.md.
- Inicializar git, hacer primer commit con mensaje
  "chore: estructura inicial del proyecto".
- NO inicializar Next.js. NO instalar dependencias. Solo estructura
  de documentación.

CRITERIO DE ÉXITO:
`git log` muestra el primer commit. `ls -la` muestra la estructura
descrita. `git status` está limpio.
```

**Commit inicial en `main`:** `chore: estructura inicial del proyecto`. (Gitflow se establece en el Bloque 2; este commit base vive en `main`.)

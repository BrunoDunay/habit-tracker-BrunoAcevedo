# Prompt 8.1 — Spec del agente implementer

**Bloque 8 · Agente implementer**

> Trabaja en la rama `feat/agente-implementer` (desde `develop`). El commit se hace al cerrar el Prompt 8.2.

```
CONTEXTO:
Voy a crear el agente `implementer` en `.claude/agents/implementer.md`.
Su trabajo: tomar una tarea concreta del plan.md, leer todo el contexto
del proyecto (spec, ADRs, AGENTS.md, sistema de diseño, plan de pruebas),
implementar la tarea, y proponer el commit.

OBJETIVO:
Escribir la spec del agente con los 4 elementos.

RESTRICCIONES:
- El implementer ejecuta UNA tarea a la vez. Nunca más de una.
- Antes de tocar código, debe:
  1. Confirmar que entendió la tarea (resumirla en sus palabras).
  2. Listar archivos que va a tocar.
  3. Esperar aprobación antes de generar código.
- Después de implementar:
  1. Listar cambios hechos.
  2. Identificar qué prueba manual del plan de pruebas valida esta
     tarea.
  3. Proponer mensaje de commit en formato del proyecto.
  4. NO ejecutar el commit. Eso lo hace el humano.
- Si se atora 2 veces en la misma tarea, debe decirlo y sugerir
  edición manual, recordando documentarla en CONTEXT.md.
- No-goals: "no avanza a siguiente tarea sin instrucción", "no
  commitea", "no modifica plan.md".
- Máximo 60 líneas.

CRITERIO DE ÉXITO:
Spec lista. Otro developer podría implementar este agente sin
contexto adicional.
```

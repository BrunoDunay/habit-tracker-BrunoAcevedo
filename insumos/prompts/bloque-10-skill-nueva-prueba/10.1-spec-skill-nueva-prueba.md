# Prompt 10.1 — Spec del skill nueva-prueba-manual

**Bloque 10 · Skill nueva-prueba-manual**

> Trabaja en la rama `feat/skill-nueva-prueba-manual` (desde `develop`). El commit se hace al cerrar el Prompt 10.2.

```
CONTEXTO:
Skill custom `.claude/skills/nueva-prueba-manual/SKILL.md`. Lo invoca
el agente QA cuando genera el plan de pruebas, y también puede
invocarlo un humano que agrega una prueba a mano.

Formato canónico de una prueba manual:
- ID correlativo
- Criterio de aceptación que cubre (referencia a spec.md)
- Precondición (estado del sistema y datos antes de la prueba)
- Pasos numerados (acción del usuario, sin ambigüedad)
- Resultado esperado (observable, verificable con sí/no)
- Estado: pendiente / pasada / fallida (default: pendiente)

OBJETIVO:
Escribir la spec del skill con los 4 elementos.

RESTRICCIONES:
- Cada prueba referencia EXACTAMENTE 1 criterio de la spec. Si cubre
  más de uno, se separa en pruebas distintas.
- Rechazar pruebas con resultado esperado no observable ("la app
  funciona bien" no es resultado esperado).
- Rechazar pasos ambiguos ("el usuario navega un poco" no es paso).
- No-goals: "no ejecuta pruebas", "no decide si una prueba pasó".

CRITERIO DE ÉXITO:
Spec lista.
```
